# エスカレーションルール

どこまで自律判断し、どこからユーザーに確認するかを定めます。
判断に迷ったとき、このファイルを確認してから動く。

## 即座にブロック・停止する（自律判断）

以下に該当する場合、投稿・処理を即停止しユーザーに報告する。実行しない。

- `sns-profile.json` の `prohibited_topics` に該当するコンテンツ
- `sns-profile.json` の `prohibited_expressions` に該当する表現
- `/content-review` の判定が `block`
- `/content-review` の `block` 判定が連続3回
- 月間投稿数が1,400件を超えた（上限1,500の93%）
- 炎上リスクのある投稿を生成してしまった
- 外部からの大量ネガティブリプライを検知

## 自律実行してよい（確認不要）

- `/content-review` が `pass` の投稿を `data/posts/` に保存
- optimizer の `confidence: high` 提案の適用
- メトリクス収集・記録（monitor の通常業務）
- 評価レポート生成（evaluator の通常業務）
- `data/eval/baseline.json` の更新
- 通常のPDCAサイクル全工程

## ユーザーに確認してから実行する

- optimizer の `confidence: medium` 提案の適用
- `sns-profile.json` のジャンル・ターゲット・ペルソナの変更
- 過去の投稿削除・修正
- X APIの設定変更
- アカウントのプロフィール変更
- 今まで試したことのない新しい戦略の本格導入
- 月間投稿残数が100件を切ったときのペース変更

## 異常検知時の対応フロー

1. 処理を停止する
2. 状況を3行以内で簡潔にユーザーに報告する
3. 推奨アクションを提示する（実行はしない）
4. `data/reports/errors/YYYY-MM-DD_HHmmss.json` にログを保存する

報告形式:
```
【要確認】{状況の一言説明}
- 何が起きたか: ...
- 停止した処理: ...
- 推奨アクション: ...
```
