# コンテンツ品質基準

「良い投稿」の定義と、品質判断の基準を定めます。
content-review / evaluator / writer が参照します。

## 良い投稿の定義

**ターゲット読者が「保存したい」「シェアしたい」と思う投稿**

インプレッションを集めるだけでなく、エンゲージメント（いいね・RT・リプライ・引用RT）を生む投稿を目指す。

## 投稿構造の基準

### フック（1行目）

読者が次の行を読みたいと思うかどうかが全て。

- 良い例: 問いかけ、驚き、共感、数字、逆説、体験談の冒頭
- 悪い例: 「〜についてまとめました」「〜を紹介します」（価値を先送りにしている）

### 本文

- 一読で理解できるか（難解な表現を避ける）
- 1投稿で言いたいことは1つに絞る
- 読者の行動・思考を変えるきっかけがあるか
- 改行を使い、スキャンしやすくする

### クロージング（CTA）

- 次のアクションが自然な流れで促されているか
- 押しつけがましくない
- 「いいねしてね」の直接訴求は最小限に

## エンゲージメント品質の優先順位

1. **引用RT** — 最高品質。文脈付きで拡散される
2. **リプライ** — コミュニティ形成。議論が生まれている証拠
3. **RT** — 量的拡散。フォロワーの外に届く
4. **いいね** — 最低品質。通過マークに過ぎない

いいねだけ多くてリプライ0の投稿は「弱い投稿」と評価する。

## content-review 判定基準

| 判定 | 条件 |
|------|------|
| `pass` | 上記基準を満たし、prohibited設定に抵触しない |
| `warn` | フックが弱い、CTAがない、文体がペルソナとずれている等（修正推奨・投稿可） |
| `block` | prohibited設定違反、誹謗中傷、根拠のない断言、文字数超過 |
